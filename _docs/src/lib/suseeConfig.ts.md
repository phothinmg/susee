## Overview

Provides helpers to locate a susee.config file in the current working directory, validate its entryPoints, and convert the user-facing SuSeeConfig into an internal BuildOptions shape consumed by the build pipeline. It exposes functions for resolving config paths, validating entries (fail-fast with colored errors), generating per-entry build descriptors with defaults, and dynamically importing the config to produce finalized BuildOptions or undefined when no config is found.

## Architecture Notes

- Configuration normalization pattern: generateBuildOptions centralizes defaulting and normalization (formats deduplicated, defaults applied, output paths computed) so callers only consume BuildOptions.
- Fail-fast validation: checkEntries prints colored errors and exits the process on invalid config (no entries, duplicate exportPath, missing entry files) rather than throwing exceptions, implying this module is intended for CLI or build-time usage.
- Dynamic config loading: finalSuseeConfig uses a runtime import of the resolved config file path and expects a default export matching SuSeeConfig, keeping loader logic decoupled from file resolution logic (getConfigPath).

## Usage Examples

### CLI/build invocation

A build script calls finalSuseeConfig(); if it returns undefined there is no susee.config file. If it returns BuildOptions, pass buildOptions.buildEntryPoints and buildOptions.outDir into the bundler. Example flow: const opts = await finalSuseeConfig(); if (opts) runBundler(opts).

## Maintenance Notes

- Entry validation is destructive (process exit) — if you want non-terminating validation, refactor checkEntries to throw errors instead of calling ts.sys.exit.
- getConfigPath checks only the current working directory for 'susee.config.ts', 'susee.config.js', and 'susee.config.mjs' in that order; it does not walk parent directories.
- generateBuildOptions expects each EntryPoint.exportPath to be '.' or start with './'. The computed outputDirectoryPath uses simple string slicing (ent.exportPath.slice(1))—ensure exportPath formats remain consistent.

### Signature

```typescript
function checkEntries(entries: EntryPoint[]): void
```

### Description

Validate an array of EntryPoint objects: ensure it's non-empty, detect duplicate exportPath values, and verify that each entry file exists; exit the process with errors on failure.


Checks that the entries array contains at least one element and exits with an error if empty. Scans entries to detect duplicate exportPath values and reports them. Resolves each entry path with ts.sys.resolvePath and checks existence using ts.sys.fileExists, exiting on any missing files.

<br>**Constraints:** Must be a non-null array, Each element should be an object with exportPath (string) and entry (string) properties (the function reads these properties), Function expects at least one entry; otherwise it will exit the process |

### Returns

**Type:** `void`

Does not return a value. On validation failures the function terminates the process using ts.sys.exit(1). If all checks pass, the function completes without returning anything.


**Possible Values:**

- Completes normally (no return value) when all validations pass
- Terminates the Node/TS host process via ts.sys.exit(1) on error conditions

### Raises

| Exception | Condition |
| --- | --- |
| `Process exit via ts.sys.exit` | Called when entries array is empty, when duplicate exportPath values are detected, or when any entry file does not exist; the function calls ts.sys.exit(1) instead of throwing an exception. |

### Side Effects

#### generateBuildOptions

![Type: Sync](https://img.shields.io/badge/Type-Sync-green)

### Signature

```typescript
function generateBuildOptions(config: SuSeeConfig): BuildOptions
```

### Description

Constructs and returns a BuildOptions object derived from the provided SuSeeConfig by normalizing each entryPoint into a BuildEntryPoint and collecting top-level build flags.


Validates and normalizes SuSeeConfig.entryPoints into BuildEntryPoint objects, applying sensible defaults (e.g., outDir='dist', format=['esm'], renameDuplicates=true). Calls checkEntries(...) for validation, computes per-entry outputDirectoryPath and flags, collects buildEntryPoints, and returns { buildEntryPoints, updatePackage, outDir }.

### Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `config` | `SuSeeConfig` | ✅ | Configuration object containing global and per-entry-point settings used to produce BuildOptions. Expected to include an entryPoints array and may include outDir and allowUpdatePackageJson among other per-entry fields.
<br>**Constraints:** Must have a property entryPoints that is iterable (array) since the function iterates over config.entryPoints, checkEntries(config.entryPoints) is invoked; if that validation fails it may propagate an error |

### Returns

**Type:** `BuildOptions`

An object (cast as BuildOptions) containing: buildEntryPoints (array of normalized BuildEntryPoint objects), updatePackage (boolean from allowUpdatePackageJson or false), and outDir (string defaulting to 'dist' when not provided).


**Possible Values:**

- { buildEntryPoints: BuildEntryPoint[], updatePackage: boolean, outDir: string }
- buildEntryPoints may be an empty array if config.entryPoints is empty
- outDir is config.outDir if provided, otherwise 'dist'
- updatePackage is config.allowUpdatePackageJson if provided, otherwise false

### Raises

| Exception | Condition |
| --- | --- |
| `Any exception thrown by checkEntries` | If checkEntries(config.entryPoints) performs validation and determines the entries are invalid, its exception will propagate from this function |

### Complexity

O(n) where n is the number of config.entryPoints (each entry point is processed once)

### Related Functions

- `checkEntries` - Called by generateBuildOptions to validate/verify config.entryPoints before processing

### Notes

- Defaulting behavior: outDir defaults to 'dist'; entry.format defaults to ['esm'] if not provided; renameDuplicates defaults to true; warning defaults to false; plugins default to an empty array.
- outputDirectoryPath is computed as outDir when exportPath is '.', otherwise outDir concatenated with exportPath.slice(1) (which assumes exportPath begins with a leading path character to be sliced).
- The function casts and returns the final object as BuildOptions with the 'as BuildOptions' assertion.
