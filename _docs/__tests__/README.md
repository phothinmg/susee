## Overview

This directory is dedicated to documentation-level tests under docs/__tests__. At the root there are no files; instead the directory is organized into focused subdirectories that hold TypeScript test files and helpers. The lcov/ subdirectory contains TypeScript files related to LCOV coverage testing (index.ts and lcov.ts) and is intended to house tests and utilities that validate or produce LCOV coverage artifacts for the docs area. The test-suites/ subdirectory contains documentation-level test suites and a root helper file (test_helpers.ts) that provides shared setup and utilities for those suites.

Together, these subdirectories provide a clear separation of concerns: lcov/ groups coverage-focused tests and validation logic, while test-suites/ groups feature-oriented docs test suites and the shared helpers they rely on. The overall role of docs/__tests__ in the larger system is to centralize test code that validates documentation behavior, coverage, and test harness integration for the docs area, enabling maintainers to run and extend docs-focused tests without cluttering the repository root with test implementation details.


### File Organization

Files are organized by test feature: coverage-related tests live in lcov/, and documentation-level test suites and helpers live in test-suites/. The root contains no files, so all actionable test code is grouped under these subdirectories to keep concerns separated and discoverable.
