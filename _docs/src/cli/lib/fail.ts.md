## Overview

This file defines and exports a single function: export function fail(message: string). The function formats the literal tag "[Error]" with magenta and the provided message with gray using the tcolor default export, writes the combined string to stderr via console.error, and immediately calls process.exit(1) to terminate the process.

Placed under src/cli/lib, this utility centralizes fatal-error reporting for command-line code paths. It depends on the @suseejs/color package for color styling and relies on Node's console and process globals to emit the message and end execution, so callers should only invoke it when an immediate CLI termination is appropriate.

## Architecture Notes

- Provides a single, side-effecting utility to standardize fatal error output and process termination in CLI code.
- Calls process.exit(1) synchronously, so it immediately terminates the Node process (no asynchronous cleanup will run after invocation).

## Usage Examples

### Main use case

From CLI code, call fail("reason") to print a colored error like '[Error] : reason' to stderr and exit the process with code 1.
