## Overview

This small Bash script computes the repository root relative to the script's location, changes into that directory, and sets Git's core.hooksPath to the .githooks directory (git config core.hooksPath .githooks). It then ensures the commit-msg hook file in that directory is executable (chmod +x .githooks/commit-msg) and prints confirmation messages including the active hooksPath.

The script enables strict error handling via set -euo pipefail so it will exit on error or on use of unset variables. It operates on the repository-level Git configuration (not global), relies on standard POSIX utilities (git, chmod, pwd, dirname, cd, echo), and expects a .githooks directory containing a commit-msg file to exist in the repository root.

## Architecture Notes

- Computes repo root relative to script location using repo_root="$(cd "$(dirname "$0")/.." && pwd)" so the script can be run from any current working directory.
- Uses git config core.hooksPath to set a per-repository hooks directory (.githooks) rather than modifying .git/hooks directly.
- Applies strict Bash execution flags (set -euo pipefail) to fail fast on errors and unset variables.

## Usage Examples

### Main use case

Run the script (e.g., ./scripts/install-hooks.sh) from anywhere; it will cd to the repo root, set core.hooksPath to .githooks, make .githooks/commit-msg executable, and print the active hooksPath. Requires git and chmod available and a .githooks/commit-msg file present.

## Maintenance Notes

- Ensure .githooks/commit-msg exists before running; the script will fail (and exit immediately) if files or commands referenced are missing due to set -euo pipefail.
- The git config command without --global modifies the repository-level config (.git/config). If a global or system-level hooksPath is desired, use --global or --system explicitly.
- On some filesystems (notably certain Windows setups), the executable bit semantics differ; chmod may not produce the expected result depending on Git/OS configuration.
