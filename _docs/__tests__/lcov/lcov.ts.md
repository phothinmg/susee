## Overview

Exports a single pure function lcovToCodecov(lcov: string) that parses an LCOV text payload and returns a Codecov-style coverage object: { coverage: { [filePath]: { [lineNumber]: hits } } }. The parser splits the input on the literal "end_of_record", extracts source filenames from lines beginning with "SF:", and captures per-line hit counts from "DA:" lines.

After extracting per-file line mappings, the function filters out test files using two repository-specific regular expressions and includes only files whose path starts with "src". The implementation performs no I/O or external calls, making it suitable for unit testing and use in build or CI pipelines.

## Architecture Notes

- Single pure utility function: no external IO, side effects, or runtime dependencies; suitable for unit testing and composition in build scripts.
- Filtering is done with two hard-coded regular expressions and a prefix check (file.startsWith('src')), which encodes repository-specific assumptions about test and source paths.
- Coverage representation uses a nested plain object: top-level 'coverage' maps file path strings to objects that map numeric line numbers to hit counts (note: object keys in JS are strings at runtime).

## Usage Examples

### Main use case

Call lcovToCodecov with the contents of an lcov.info file (string). The function returns { coverage } where coverage[filePath][lineNumber] = hits. Example flow: read lcov.info as string -> lcovToCodecov(lcovString) -> write JSON to disk or post to an API.

## Maintenance Notes

- The parser expects DA lines formatted as 'DA:<line>,<hits>'; it uses simple split and Number() conversion without validation—malformed DA lines may produce NaN keys/values.
- Test-file filtering is repository-specific: update the two regexes or the 'src' prefix check if your project places sources/tests in different directories or uses different file name conventions.
- Line numbers are stored as numeric keys in a plain JS object; when serialized to JSON, object keys become strings. If a mapping that preserves numeric keys is required, consider using arrays or Maps and a stable serialization strategy.

### Signature

```typescript
function lcovToCodecov(lcov: string)
```

### Description

Parses an LCOV coverage string and returns a Codecov-style JSON object with per-file coverage, excluding test and non-src files.


Splits the LCOV string into records using the literal token 'end_of_record' and parses each record for SF: (source file) and DA: (per-line) lines. Builds a coverage map for files whose paths start with 'src' and that do not match configured test-file patterns, mapping line numbers to hit counts, and returns { coverage }. 

### Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `lcov` | `string` | ✅ | A string containing LCOV-formatted coverage data (multiple records separated by the literal token 'end_of_record').
<br>**Constraints:** Must be a string containing LCOV content, Records are expected to include lines starting with 'SF:' for source file and 'DA:' for per-line data |

### Returns

**Type:** `{ coverage: { [file: string]: Record<number, number> } }`

An object with a single key 'coverage' whose value is a map from source-file path to an object that maps line numbers to hit counts. Only files that start with 'src' and do not match the test-file regexes are included.


**Possible Values:**

- A coverage map containing entries for each included source file: { coverage: { 'src/...': { 10: 1, 11: 0, ... }, ... } }
- An object with an empty coverage map if no records match the inclusion filters: { coverage: {} }

### Complexity

O(n) where n is the number of lines in the LCOV input

### Notes

- Filters out files matching /^.*\.test\.ts$/ and /^test\/(?:esm|ts)\/.*(?:\.js|\.ts)$/.
- Only includes files whose path starts with 'src'.
- Per-line DA parsing uses split(',') and Number; non-numeric values will become NaN in the resulting keys/values if present.
- Records are split by the literal string 'end_of_record' and trimmed; trailing/leading whitespace is removed from lines.
