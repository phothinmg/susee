## Overview

This directory groups test-case fixtures and configuration modules for the get_config test-suite under docs/__tests__. At the root there are no files; instead the directory is organized into per-scenario subdirectories. Several subdirectories (duplicate, empty_entries, entry_not_found, ok) each contain a single TypeScript configuration file named susee.config.ts at their root which serves as the primary artifact for that test-case. The not_found subdirectory contains no root files and instead splits its code into further nested folders (for example src/), housing the runtime configuration module and helper code used by the not_found tests.

Collectively these subdirectories represent concrete scenario fixtures used by higher-level repository or documentation tests to exercise get_config behavior: the 'ok' case provides a working configuration module and implementation, 'duplicate' contains fixtures that simulate duplicate-configuration scenarios, 'empty_entries' provides a configuration representing empty entries, and 'entry_not_found' offers assets and helper code for missing-entry scenarios. This layout allows tests to import or require a scenario's susee.config.ts (or its nested module in not_found) and run deterministic assertions against get_config handling for each variant.


### File Organization

The directory is organized by test-case variant: each subdirectory corresponds to a named scenario and contains the files required for that scenario. Root contains no files; scenario-specific files (notably susee.config.ts) live at the root of their respective subdirectory or within that subdirectory's internal source tree (as in not_found). This makes it easy to add new scenarios as new subdirectories without changing other fixtures.

## Architecture Notes

- Test fixtures are scenario-scoped: each subdirectory encapsulates the configuration and supporting code for a single get_config test scenario, enabling isolated and maintainable test fixtures.
- The directory intentionally contains no root-level files; organization is done per-variant to keep fixtures explicit and avoid cross-test coupling.
