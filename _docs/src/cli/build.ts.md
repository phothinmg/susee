## Overview

Exports an async function cliBuild that orchestrates a simple CLI build flow: it loads configuration via finalSuseeConfig(), exits the process with ts.sys.exit(1) if the config is missing (printing a colored error), and on success constructs a Compiler with the loaded BuildOptions and calls await compiler.compile(). The run is wrapped with console.time/console.timeEnd using a cyan "[Build]" label. Compilation errors are not caught here and will propagate.

## Architecture Notes

- Thin CLI wrapper pattern: this module delegates actual work to project internals (finalSuseeConfig and Compiler) and contains only orchestration and user-visible messages.
- Uses simple synchronous control flow inside an async function: await finalSuseeConfig(), conditional exit on missing config, then instantiate Compiler and await compile().
- Relies on TypeScript runtime (ts.sys.exit) for process termination rather than throwing an error or returning an exit code.

## Usage Examples

### Main use case

Call cliBuild() from a top-level CLI script (or import and invoke it) to perform a project build: the function loads the final configuration via finalSuseeConfig(), logs and exits if missing, then creates a Compiler with the typed BuildOptions and runs compiler.compile(), showing timed output labeled '[Build]'.

## Maintenance Notes

- cliBuild performs a type assertion (buildOptions as BuildOptions); ensure finalSuseeConfig() returns the expected shape to avoid runtime type mismatches inside Compiler.
- Missing configuration results in immediate process termination via ts.sys.exit(1); callers that embed this function should be aware it does not throw but exits the process.
- No try/catch around compiler.compile(): compilation errors will propagate. If you need graceful CLI error reporting, wrap the call and handle/reformat errors here.
