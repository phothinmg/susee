## Overview

This Bash script provides an interactive commit helper for repository-root workflows. It lets a user choose a commit type from a numbered menu, enter a commit message, run npm tasks (lint, fmt, coverage), stage changes, commit with a message prefixed by the selected type, and push to the current branch.

The script enforces strict shell options (set -euo pipefail), resolves and cds to the repository root so commands run from project root, and aborts the commit/push flow if any npm task or git command fails.

## Architecture Notes

- Interactive CLI: Uses Bash builtins (PS3, select, read) to provide a simple interactive prompt for selecting commit types and reading the commit message.
- Pre-commit task enforcement: Runs npm-run scripts (npm run lint, npm run fmt, npm run coverage) sequentially; because set -euo pipefail is enabled, any failure aborts the commit and push flow.
- Repository-root execution: Resolves repo root with repo_root="$(cd "$(dirname "$0")/.." && pwd)" and cds into it so commands operate from the project root regardless of invocation directory.

## Usage Examples

### Main use case

Run ./scripts/commit.sh. Select a commit type from the numbered list (PS3 prompt), enter the commit message when prompted, let the script run npm run lint, npm run fmt, and npm run coverage. If all tasks pass, the script will git add ., git commit -m "<TYPE> : <message>", and git push origin <current-branch>.

## Maintenance Notes

- Because the script runs git add . it stages all working-directory changes; if you want to only commit staged changes modify or remove that line to avoid unintentionally committing unrelated files.
- The script uses npm run lint/fmt/coverage and will abort on any non-zero exit due to set -euo pipefail. Ensure those npm scripts are defined in package.json and succeed in your environment before running the script.
- Commit message format uses commit_message="$TYPE : $message" (there is a space before and after the colon). If you have a conventional commit parser that expects "type: message" without surrounding spaces, update this line accordingly.
