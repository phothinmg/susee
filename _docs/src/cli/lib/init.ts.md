## Overview

This module exports an async init() function designed for a CLI to create a Susee config file at the project root. It prompts the user to determine whether the project uses TypeScript; for TypeScript it writes susee.config.ts from an embedded TypeScript template. For JavaScript it reads package.json to detect pkg.type and chooses susee.config.mjs (ESM) or susee.config.js (CommonJS) and writes a JavaScript template. If a config file already exists at the resolved path it is removed before writing. The implementation uses Node built-ins (fs, path, process) and readline/promises for prompting, and uses @suseejs/color.tcolor.cyan for colored CLI output.

## Architecture Notes

- Simple linear CLI flow: prompt → determine template → resolve path → remove existing file (if any) → write template → log success.
- Decision logic for JS config filename relies on package.json's top-level "type" field; no fallback or error handling if package.json is missing or invalid.
- File templates are embedded as module-level string constants (tsFileText and jsFileText) to avoid external template files.

## Usage Examples

### Main use case

Invoke the exported init() from a CLI entrypoint. The script prompts: 'Is TypeScript project(y/n) : '. If you answer 'y' (or press Enter), it writes susee.config.ts with the TypeScript template. If you answer 'n', it reads package.json to decide between susee.config.mjs and susee.config.js and writes the JavaScript template to the project root. Existing config file at the same path is deleted before writing.

## Maintenance Notes

- No error handling around reading package.json: fs.promises.readFile in getPackageType() will throw if package.json is missing or unreadable; callers should catch errors if running in environments without package.json.
- The prompt treats empty input as affirmative for TypeScript (empty string -> TypeScript).
- When writing JavaScript config, logic assumes only two outcomes from package.json ("module" → esm, otherwise commonjs). If more nuanced handling is required (workspaces, monorepos), update getPackageType().
