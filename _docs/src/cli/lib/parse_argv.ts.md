## Overview

This module implements command-line parsing for a build tool. It defines the CliOptions and CliBuildOptions shapes and provides helpers (isFile, isEmptyObject, parseBooleanFlag). parseArgs converts an argv array into a partial CliOptions, accepting both --flag=value and separate-token forms, detecting the first file-like token as the entry, validating flags, and calling fail(...) on invalid input.

getDefaultOptions fills missing values with sensible defaults (outDir='dist', format='esm', rename=true, allowUpdate=false, minify=false, warning=false) and returns a fully typed CliBuildOptions including a plugins array.

## Architecture Notes

- Imperative single-pass argv parsing: parseArgs iterates argv by index and conditionally increments the index when it consumes a separate next token; it also accepts inline "--flag=value" syntax.
- Validation-first approach: parseBooleanFlag and conditional checks inside parseArgs call fail(...) immediately on invalid input to stop processing and surface a clear error.
- Separation of concerns: parseArgs produces a partial CliOptions (possibly with undefined fields), and getDefaultOptions is responsible for filling defaults, typing, and producing the final CliBuildOptions including plugin typing.

## Usage Examples

### Main use case

Call parseArgs(process.argv.slice(2)) to produce a CliOptions object. Then pass that result to getDefaultOptions(parsed) to obtain a fully populated CliBuildOptions used by the build process. Example flows handled: first token can be an entry file (e.g., "src/index.ts"), flags can be provided as "--outdir=dist" or "--outdir dist", and boolean flags like "--rename" (implicit true) or "--rename false" (explicit false).

## Maintenance Notes

- Boolean flags accept implicit presence (sets true), inline values ("--flag=true"/"--flag=false"), or next-token values; parseBooleanFlag only accepts exact strings "true" or "false" and will call fail on other values.
- The isFile extension list is hard-coded: [".js", ".ts", ".mts", ".mjs", ".cjs", ".cts"]. Update this list if additional entry file extensions are needed.
- parseArgs treats any token starting with "--" as a flag; ensure callers strip node and script paths (e.g., process.argv.slice(2)). The function will call fail if no entry is provided or if required flag values are missing.
