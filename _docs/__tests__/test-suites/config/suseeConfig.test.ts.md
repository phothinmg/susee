## Overview

This test file uses Node's built-in test runner (node:test) to verify finalSuseeConfig from src/lib/suseeConfig.js across multiple filesystem fixtures. Tests change process.cwd() to fixture directories under __tests__/test-suites/config/get_config/*, call finalSuseeConfig (or helpers that assert process exit), and assert outcomes or returned configuration objects.

Each test restores the original cwd in a finally block. The suite uses node:assert for assertions, a test helper (exitWithCodeOneAndMessage) to intercept and validate process exit behavior, and TypeScript's ts.sys.resolvePath to compute deterministic fixture paths.

## Architecture Notes

- Tests rely on switching process.cwd() to filesystem fixtures and restoring it in finally blocks to isolate each scenario.
- Node's built-in test runner (node:test) and node:assert are used instead of external test frameworks; a helper function intercepts/validates process exit behavior rather than letting tests actually terminate the process.
- TypeScript's runtime sys.resolvePath is used to build deterministic paths to test fixtures.

## Usage Examples

### Main use case

Run the test suite with Node (e.g., node --test). Each it block changes process.cwd() to a fixture folder, calls finalSuseeConfig (or exitWithCodeOneAndMessage), asserts expected behavior (exit message or returned config), and finally restores the original cwd. Expected outcomes include specific exit warnings for invalid configs and a structured BuildEntryPoint object for the OK fixture.

## Maintenance Notes

- Because tests mutate process.cwd(), ensure any future tests restore cwd in finally blocks to avoid cross-test pollution.
- Fixtures under __tests__/test-suites/config/get_config/* are required for these tests; moving or renaming them will break the suite.
- Tests depend on exitWithCodeOneAndMessage helper to validate process exit behavior; changes to how process exits are intercepted may require updating that helper and these tests.
